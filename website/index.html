Capability Map

功能按场景分组，不把信息挤成一团。

正式版突出监控、风扇、电池、性能模式、GPU 模式和灯效。GPU 遥测只读；不提供 GPU 超频、VF 曲线、GPU 锁频、游戏 FPS 检测或硬件实时监控叠加层。

先告诉用户机器现在怎么跑。

温度、功耗、频率、风扇、电池和扩展传感器统一显示。能读到真实来源就展示来源，读不到就明确降级，不伪造状态。

  • ThermalInfo、LibreHardwareMonitor、EC 多来源兜底。
  • 风扇支持 RPM 与百分比切换，第三、第四风扇按机型映射。
  • AMD 平台传感器和 PCH 温度按实际可用性展示。

常用控制集中，高风险操作给边界。

风扇曲线、全速模式、性能模式、电源计划、电池充电上限、GPU 模式和触摸板快捷键放在固定入口里。

  • CPU / GPU 风扇曲线、风扇校准和一键全速。
  • 性能模式、电源计划、系统能效与电池健康。
  • GPU 模式切换、触摸板开关和固定快捷键。

SteelSeries 灯效保留独立入口。

灯光页不是旧设置滑块。它面向 SteelSeries 键盘，覆盖区域、亮度、速度、调色板、HID 通道和 GameSense 兜底。

  • SteelSeries KLC / HID 通道检测与诊断。
  • 预设灯效、区域映射和部分逐键路径。
  • 原生驱动栈检测、恢复和问题报告入口。

出了问题，用户要知道怎么反馈。

日志、诊断报告、MSI 中国区支持页、FAQ 和机型兼容说明都应该在发布页有入口，减少“打不开”和“灰色不可用”的无效沟通。

  • 本地错误日志记录硬件操作失败上下文。
  • MSI 中国区官网支持页跳转，不在软件内爬驱动列表。
  • 反馈模板要求机型、BIOS / EC、驱动版本和复现步骤。

Setup Flow

安装流程要把坑提前讲完。

面向小白用户时，发布页要把设备要求、管理员权限、依赖、冲突软件和校验方式放在下载按钮附近，而不是藏在长 README 里。

01

确认设备

Windows 10 / 11 x64，MSI 笔记本，已安装显卡驱动和 MSI 相关平台驱动。

02

校验文件

下载后对照发布页 SHA256。校验不一致就不要运行，重新从 Release 或国内镜像下载。

03

管理员启动

EC、风扇、MSR、驱动恢复等底层功能需要管理员权限，权限不足时应显示明确提示。

04

避免冲突

不要同时运行 MSI Center、Dragon Center、Afterburner 等可能接管风扇或 GPU 的工具。

Free, Local, Closed Source

为什么选择闭源免费。

M-Control 是免费工具，但当前版本不公开项目代码。为了适配 MSI 笔记本的 EC、风扇控制器、键盘灯效和 GPU 模式等底层能力，部分实现涉及未公开接口、私有协议适配和逆向分析资料。

如果把这些实现细节公开，可能给作者和使用者带来不必要的知识产权或合规风险。为了让项目能长期维护和更新，当前选择闭源免费发布。

不做遥测

不收集用户数据，不上传硬件信息，不读取个人文件。

本地优先

核心硬件控制在本机执行；网络访问只用于用户主动打开的支持页或版本信息。

欢迎验证

发布文件提供 SHA256。用户可以用杀毒软件扫描，也可以放进虚拟机或测试环境验证。

尊重选择

如果你无法接受闭源工具，可以先不使用，等更多实机反馈和发布记录。

Compatibility

兼容性要坦诚，不能包装成万能工具。

M-Control 会访问 EC、MSI WMI、UEFI、NVAPI / NVML、SteelSeries HID 等接口。不同机型、BIOS、EC 固件和驱动栈都会影响功能可用性。

提交兼容反馈
适合

支持 EC 风扇控制、MSI WMI、独显模式切换和 SteelSeries 键盘的 MSI 机型。

不保证

非 MSI 设备、特殊 BIOS / EC 固件、驱动异常或硬件通道缺失的设备。

需谨慎

风扇曲线、GPU 模式切换、电池限制和底层硬件访问应先了解风险；正式版 GPU 遥测仅为只读。

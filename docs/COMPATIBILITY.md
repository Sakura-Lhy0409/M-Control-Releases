# M-Control 兼容性说明

M-Control 主要面向 MSI 微星笔记本。兼容性由机型、主板平台、BIOS、EC 固件、驱动版本和实际硬件通道共同决定，不能只按系列名称推断。

## 状态定义

| 状态 | 含义 |
| --- | --- |
| ✅ 已验证 | 维护者或用户提供了对应功能的实机结果 |
| ⚠️ 部分验证 | 核心功能可用，但存在明确限制或未覆盖模块 |
| ❓ 待验证 | 尚无足够实机证据 |
| ❌ 不支持 | 已确认缺少必要硬件或固件能力 |

“已验证”只针对记录的 BIOS、EC、驱动和功能范围，不代表同系列所有配置完全一致。

## 正式版功能范围

- 监控：CPU / GPU 温度、频率、功耗、利用率、风扇和电池状态。
- 风扇：曲线、全速、校准和 RPM / 百分比显示。
- 性能：情景模式、Windows 电源计划和系统能效。
- GPU 模式：核显、混合、独显，取决于 MUX 和固件能力。
- 电池：充电上限、健康、容量和循环信息。
- 灯效：兼容 SteelSeries 设备的灯效和 HID 控制。
- 快捷键与 OSD：取决于 WMI、EC 和键盘事件通道。

GPU 遥测是只读功能。正式版不包含 GPU 超频、VF 曲线、GPU 锁频、游戏 FPS 检测或硬件实时监控叠加层。

## 影响兼容性的关键条件

### BIOS 和 EC

不同 BIOS / EC 版本可能改变寄存器、WMI 返回值、风扇曲线和快捷键事件。更新 BIOS 前后都应记录版本和行为，不要假定更新一定改善兼容性。

### MSI 平台驱动

部分能力需要可用的 MSI ACPI WMI Provider / MSIWMIACPI2 或相关平台组件。请优先从 MSI 对应机型支持页获取驱动，不要从不明第三方下载。

### GPU 与 MUX

- NVIDIA 只读遥测需要兼容的 NVIDIA 驱动及可用的 NVAPI / NVML 等接口。
- 没有独显或没有 MUX 能力的设备不会提供完整 GPU 模式切换。
- AMD / Intel GPU 的可用监控字段可能与 NVIDIA 不同。

### 风扇和传感器

风扇数量、RPM、PWM、ThermalInfo、PCH、系统功耗和扩展传感器都可能因机型而异。没有可信读数的项目会隐藏或显示不可用。

### SteelSeries 键盘

灯效能力取决于 PID、固件、HID 描述符、现有驱动和通道。M-Control 不承诺发布包附带第三方 SteelSeries 驱动；未匹配设备时灯效会降级或不可用。

## 通常不支持的环境

- 非 MSI 笔记本：大多数硬件控制不可用，部分通用监控可能仍能读取。
- MSI 台式机：项目按笔记本 EC / WMI 路径设计，不作为支持目标。
- 很老或特殊固件机型：可能缺少当前使用的 WMI 或 EC 能力。
- 多个控制中心同时运行：设置可能相互覆盖，无法作为可靠兼容性结果。

## 提交兼容性报告

请前往 [GitHub Issues](https://github.com/Sakura-Lhy0409/M-Control-Releases/issues)，标题建议使用：

```text
[兼容性反馈] 完整机型 - BIOS 版本
```

请提供：

1. 完整机型、CPU、GPU 和键盘类型。
2. Windows、BIOS 和 EC 版本。
3. MSI 平台驱动、芯片组和显卡驱动版本。
4. 每项已验证功能及其结果。
5. 清晰的复现步骤、截图和必要日志。
6. 是否同时运行 MSI Center、Dragon Center 或其他硬件控制软件。

诊断附件可能包含设备标识、路径或系统信息。上传前请自行检查并移除不希望公开的内容。

# M-Control v1.3.16

发布日期：2026-07-18

本版本补齐监控数据源的自动恢复与熔断机制，避免一次传感器异常或永久卡住的读取任务让相关数据只能依靠重启 M-Control 恢复。

## 主要更新

### 监控自动恢复

- LibreHardwareMonitor 打开或刷新失败后不再永久禁用；保留上一份有效快照，并按 `2s/5s/15s/60s` 退避重开。
- 窗口从托盘恢复时允许 LibreHardwareMonitor 立即执行一次恢复探测。
- ThermalInfo 使用温度、风扇 RPM 和 PWM duty 三条核心读取判断连接健康；连续三次全部无响应后释放旧连接并重新建立。
- 单个可选 ThermalInfo 命令不支持或返回空值时不会误触发整条连接重建。

### Provider 卡死保护

- GPU、Intel IPF 与 LibreHardwareMonitor 慢采样任务新增开始时间、失败计数、冷却时间和 hung 状态。
- 普通失败按 `2s/5s/15s/60s` 退避后执行单次恢复探测。
- 单个读取超过 8 秒仍未完成时停止等待，但保留原任务且不创建第二个任务，避免后台线程不断累积。
- 原卡死任务若后来完成，会自动消费结果并关闭熔断；永久不返回时继续使用缓存、EC、ThermalInfo 等其他来源降级显示。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 此前遇到监控数据长期停在问号、空值或旧值的用户，升级后无需额外清理配置。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

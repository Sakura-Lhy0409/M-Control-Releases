# M-Control v1.3.0

发布日期：2026-07-16

本版本建立针对少数用户偶发退出、卡死或界面异常的完整诊断证据链。当前尚无证据表明存在所有机型都会触发的统一崩溃根因，因此本版本重点是让下一份客户诊断包能够给出故障模块、异常码和退出前阶段，而不是把未知问题误报为已经修复。

## 主要更新

- 新增原子运行会话标记，记录应用版本、PID、启动时间、最后生命周期阶段和最后托管异常。
- 正常退出、升级退出和有意的强制退出会明确写入完成状态；真正非正常终止会在下次启动时保留为上一异常会话。
- 诊断包新增最近 14 天的 Windows `Application Error`、`Application Hang`、`.NET Runtime` 和 `Windows Error Reporting` 事件。
- Windows 事件只收集消息中明确包含 `MControl.exe` 或 `M-Control` 的条目，最多 100 条，不导出无关应用事件或个人文件。
- 诊断包同时包含当前运行会话与最近一次非正常退出标记，可区分托管异常、WebView2 进程问题、原生 DLL/驱动故障、应用卡死和外部终止。
- 保留原有 UI 异常捕获、后台任务异常记录和 WebView2 子进程自动恢复逻辑。

## 发布包

- `MControl_v1.3.0_Setup.exe`：普通安装版。
- `MControl_v1.3.0_Setup_WebView2_Fixed.exe`：内嵌固定版 WebView2 Runtime 的安装版，适合系统 WebView2 环境异常的机器。
- `MControl_v1.3.0_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

## 后续反馈

发生退出、卡死或界面异常后，请重新打开 M-Control，在“关于与反馈”中生成诊断包并上传。诊断包中的 `windows_crash_events.txt` 与 `runtime/last-unclean-session.json` 是本次定位的关键文件。

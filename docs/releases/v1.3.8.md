# M-Control v1.3.8

发布日期：2026-07-17

本版本完成 MSI 官方运行时残留的彻底清理，统一使用 M-Control 自研 MSI ACPI 通道，并修正迁移过程中电池充电上限优先级被意外改变的问题。

## 主要更新

- 删除旧 `MsiWmiInterface`、NBFoundation、MSIService `simple` pipe 和对应启动探测、诊断残留。
- 通用 EC 访问统一为自研 MSI ACPI、PawnIO、WinRing0 三层候选，不再打包或调用 MSI 官方运行时文件。
- 情景模式和 Win/Fn 等已验证 DataCommand 能力继续优先使用自研 MSI ACPI。
- 电池健康、充电上限 `0xD7` 和电池校准适配器保留已经校准的 PawnIO/端口优先例外。
- 修复 v1.3.7 中 `GetBatteryChargeLimit` / `SetBatteryChargeLimit` 被封装内部 ACPI-first 顺序架空的问题。
- 新增方法内部顺序回归门禁，避免以后只检查外层调用名而漏掉候选顺序变化。

## 发布包

- `MControl_v1.3.8_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.8_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.8` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 验证结果

- 117 项 Node 静态与行为测试全部通过，包括真实 Inno 覆盖升级行为测试。
- 191 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- M-Control 独立运行 80 秒句柄采样为 `696 -> 694`，未发现启动后快速句柄泄漏。
- 正式安装包与便携包的 SHA256 已与发布清单逐项核对。
- 发布目录和安装包中未发现 `MsiWmiInterface`、`MSIService.exe`、`MSIWMIACPI2.dll`、NBFoundation 或 `simple` pipe 残留。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 使用电池充电上限的用户建议升级，以恢复既定的 PawnIO/端口优先读取与写入顺序。
- Win/Fn、情景模式及其他 DataCommand 功能仍保持自研 MSI ACPI 优先，没有改回 MSI 官方组件。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

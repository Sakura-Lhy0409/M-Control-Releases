# M-Control v1.3.3

发布日期：2026-07-17

本版本汇总 `v1.3.0` 之后的 Mystic Light 官方报文、PRISM 实时静态写入和风扇曲线状态修复。

## 主要更新

- Mystic Light 4/24/25/26 区灯效序列化改为按设备类型和真实 PID 生成，对齐 MSI Center 的颜色 stop、效果 ID、周期、波长、全区选择和发送顺序。
- 修复 4 区 Wave、Bright Wheel、Dance 等灯效与官方实现不一致的问题，并补充 `PID 1606` Wave 特例。
- PRISM 的关闭、纯色和逐键静态实时效果统一使用当前机型 descriptor 现场生成整帧 direct-write 报文，不再追加旧 `write-all` 或退化为逐键 RGB burst。
- PRISM 实时静态路径继续按目标机型键位和 ALC 分区生成，不包含 Titan/Raider 采集帧，也不改变 Legacy 固件持久灯效。
- CPU/GPU 风扇曲线改为分别保存曲线、硬件边界、编辑状态和配置名称，修复切换页签后显示混合曲线或名称与内容不一致的问题。
- 修复“恢复启动曲线”后切换 CPU/GPU 页签，GPU 曲线仍是启动预览但名称错误变回用户配置的问题。
- 手动风扇曲线的启动恢复开关改为后端持久化，不再完全依赖 WebView 启动时一次模式同步；切回自动模式会明确关闭启动写回。

## 发布包

- `MControl_v1.3.3_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.3_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.3` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 验证结果

- 114 项 Node 静态与行为测试全部通过。
- 177 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- 新增 CPU/GPU 页签状态行为测试，覆盖首次切换、异步硬件读回和启动曲线预览往返切换。
- Mystic Light 包含 28 个生产构建器黄金向量；PRISM 实时静态路径包含五字节/七字节 KLC 帧和 1/3/4 区 ALC 黄金向量。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。
- 使用 Mystic Light 或 PRISM 机型的用户，升级后建议重新应用一次当前灯效；使用手动风扇曲线的用户应确认界面仍处于手动模式。

# M-Control v1.3.9

发布日期：2026-07-17

本版本修正 MSI Mystic Light 分区键盘的 HID 发送层。此前报文字节和发送顺序已对齐 MSI Center，但底层仍复用通用 HID 兜底；部分四分区机型应用灯效约 15 秒后失败，随后灯效页无法继续取得设备状态。

## 主要更新

- Mystic Light 4/24/25/26 区键盘改用 MSI 官方同款 Feature Report 传输边界。
- 每条命令只调用 `SetFeature`，随后使用 report ID `01` 执行 `GetFeature` 确认，不再尝试 `SetOutputReport` 或 `WriteFile` 兜底。
- 四分区每组“选区 + 效果”共用同一个 HID 句柄，确认成功后才继续下一组；最终保存使用独立会话。
- 24/25/26 区动态效果、逐区静态以及顶部保存/取消入口同步迁移到专用 Feature session。
- 写入失败时记录 VID/PID、命令前三字节和 Win32 错误码到 `lighting-host.log`，便于诊断具体失败命令。
- SteelSeries Legacy / Prism、单色键盘和其他 HID 写入路线保持不变。

## 发布包

- `MControl_v1.3.9_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.9_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.9` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

- 117 项 Node 静态与行为测试全部通过。
- 191 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- Mystic Light 门禁确认应用、逐区静态、保存和取消命令不再回落到通用 HID 发送器。
- 正式安装包与便携包的 SHA256 已在本地重新计算。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 使用 MSI Mystic Light 四分区或多分区键盘的用户建议升级。
- 如果应用灯效仍失败，请重新生成诊断包；新版日志会直接记录失败命令和 Win32 错误码。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

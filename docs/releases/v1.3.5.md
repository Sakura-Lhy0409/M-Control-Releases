# M-Control v1.3.5

发布日期：2026-07-17

本版本修复部分新机型成功应用手动风扇曲线后，重启仍恢复默认曲线的问题。

## 主要更新

- 修复 `MS-15M3`、`E15M3IMS.117` 等包含字母的 MSI 主板代码无法生成稳定机型键的问题。
- 修复上述机型无法创建 `fan-curve-state.json`，导致启动恢复没有 CPU/GPU 保存目标的问题。
- 手动曲线成功应用后，由后端直接启用启动恢复，不再只依赖前端模式同步时序。
- 机型键仍不可用时写入明确诊断日志，避免状态保存被静默跳过。
- 新增数字与字母数字 MSI 板号回归测试，保留 `MS-1824` 等旧式纯数字板号兼容性。

## 发布包

- `MControl_v1.3.5_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.5_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.5` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 验证结果

- 115 项 Node 静态与行为测试全部通过。
- 182 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- `MS-15M3`、`E15M3IMS.117`、`MS-1824` 和 `E1824IMS.30B` 机型键样本全部通过生产代码回归测试。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 受风扇曲线问题影响的用户升级后，需要重新进入手动模式，分别应用一次 CPU 和 GPU 风扇曲线，然后重启电脑验证自动恢复。
- 不需要为了本次修复主动运行独立诊断程序；只有升级并重新应用曲线后仍然失败时，才重新采集诊断包。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

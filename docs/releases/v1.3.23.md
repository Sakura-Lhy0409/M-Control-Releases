# M-Control v1.3.23

发布日期：2026-07-22

本版本修复安装 SteelSeries GG 后，PRISM 实时灯效可用但所有“（固件）”灯效均无法应用的问题。旧逻辑把物理 KLC 固件报文发给了 GG 暴露的虚拟 KLC；现在固件部署会使用同机物理 KLC，实时灯效仍保持原 native/虚拟路线。

## 主要更新

- 当 GG native/虚拟 KLC 与物理 FW-basic KLC 同时存在时，固件持久灯效自动改用物理 KLC 端点。
- 修复覆盖当前全部 native/虚拟 KLC 产品和已实现的 `PID_1122/113A` 物理固件端点，不按单一机型或单个虚拟 PID 特判。
- 写入前核对物理端点能力和有效 descriptor 身份，避免多设备场景串路。
- 固件亮度、KLC 程序和 ALC 程序使用同一物理组合，不再重复写入附属 ALC。
- PRISM 实时效果、设备展示和已知状态继续使用原 native/虚拟 KLC，不改变实时灯效性能路线。
- 固件物理写入失败时显示实际目标 PID，便于后续诊断。

## 发布包

- `MControl_v1.3.23_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.23_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.23` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 同时安装 SteelSeries GG 的用户无需卸载 GG；M-Control 会按效果类型自动区分虚拟实时端点和物理固件端点。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

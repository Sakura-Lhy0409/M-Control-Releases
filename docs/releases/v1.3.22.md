# M-Control v1.3.22

发布日期：2026-07-20

本版本补齐 LightingHost 的启动所有权语义。v1.3.21 已避免主界面退出时重放灯效，但重新打开 M-Control 时，启动恢复仍会向已经运行的后台宿主发送 resume，导致当前灯效再次部署并从头播放。

## 主要更新

- 重新打开主界面时不再向已经运行的 LightingHost 发送 resume、reload 或 apply 命令。
- 已开启“退出时保持灯效”且后台宿主仍存活时，重新打开 M-Control 只读取现状，动态灯效会连续运行。
- 真正的 Windows 冷启动或 LightingHost 已停止后，易失实时灯效仍会按原策略恢复一次。
- 冷启动只确保 LightingHost 存在，由新宿主自身完成一次恢复，不再叠加第二条恢复命令。
- 节能模式的固件 blackout 守卫保持不变，仍优先停止灯效宿主。

## 发布包

- `MControl_v1.3.22_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.22_Portable.zip`：普通便携版，完整解压后运行。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 需要退出 M-Control 后继续播放动态灯效时，保持“退出时保持灯效”开启即可；退出和重新打开主界面都不会重新播放当前灯效。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

# M-Control v1.3.21

发布日期：2026-07-20

本版本修正“退出时保持灯效”的所有权语义。灯效本来就由独立 LightingHost 执行，主界面退出时只应决定是否关闭这个现有宿主；此前退出流程却向宿主发送 reload，导致当前灯效在退出瞬间重新部署并从头播放。

## 主要更新

- “退出时保持灯效”改为独立开关，不再与当前灯效、设备路线、随 Windows 启动或其他设置联动。
- 选择或应用“（固件）”效果不再禁用、清空该偏好，之后切回实时效果无需重新设置。
- 开启退出保持后，从托盘退出只保留现有 LightingHost，不再发送 handoff、reload 或恢复命令，避免灯效重播。
- 关闭开关只保存偏好，不会在主界面仍运行时立即终止灯效宿主；真正退出时才停止。
- 节能模式关灯守卫和“彻底退出”仍会明确停止 LightingHost。
- 易失静态实时效果的开机恢复逻辑保持不变。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 需要退出 M-Control 后继续播放动态灯效时，在设置页开启“退出时保持灯效”，再从托盘菜单退出。
- 不需要后台灯效时关闭该开关，LightingHost 会在真正退出时停止。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

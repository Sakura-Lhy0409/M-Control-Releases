# M-Control v1.1.14

发布日期：2026-07-15

M-Control v1.1.14 修复切换 Windows 系统能效模式时可能出现 AC、DC 注册表访问被拒绝的问题，并纠正失败提示仍声称“已写入”的误导文案。

## 主要更新

### 系统能效模式

- 删除 `HKLM\SYSTEM\CurrentControlSet\Control\Power\User\PowerSchemes` 的直接写入兜底。
- Windows 11 继续使用官方 `PowerSetUserConfiguredACPowerMode` / `PowerSetUserConfiguredDCPowerMode` 同步 AC、DC 模式。
- 不支持独立 AC/DC API 的旧系统改用 `PowerSetActiveOverlayScheme`，只切换当前正在使用的供电状态。
- P/Invoke 参数改为与 Windows SDK 一致的 `ref Guid`。
- 所有切换继续经过硬件写入策略门禁，并由 Windows 电源管理接口负责状态通知。
- 失败提示改为“系统能效模式切换到某模式失败”，不再错误显示“已写入”。

Windows 10 没有公开的独立 AC/DC 用户电源模式写接口，因此本版本不会通过修改受保护注册表强行同步另一种供电状态。插拔电源后，由 Windows 自身保存的对应供电配置决定实际模式。

## 发布包

- `MControl_v1.1.14_Setup.exe`：普通安装包，包含系统 WebView2 与 PawnIO 前置条件处理。
- `MControl_v1.1.14_Portable.zip`：普通免安装便携版，使用系统已安装的 WebView2 Runtime。
- `SHA256SUMS.txt`：两个正式产物的 SHA256 校验清单。

`v1.1.14` 的补丁位不为 `0`，属于 hotfix，因此不提供内嵌 WebView2 Fixed Version Runtime 的大安装包。Fixed 安装版仅随补丁位为 `0` 的基线版本提供。

## 升级提示

- 从 `v1.1.0` 或更高版本可直接运行 `MControl_v1.1.14_Setup.exe` 覆盖安装。
- 遇到 AC/DC 注册表访问被拒绝的用户，应升级后在原机器重新切换一次系统能效模式。
- 早于 `v1.1.0` 的安装版建议先卸载旧版，再安装本版本。
- 便携版必须完整解压后运行；需要内嵌 WebView2 的用户应使用最近一个补丁位为 `0` 的基线版本 Fixed 安装包，或安装系统 WebView2 Runtime 后使用本 hotfix。

# M-Control v1.3.6

发布日期：2026-07-17

本版本修复少数机器在 WebView2 首次初始化失败后，同进程自动重试反而导致整个控制台无响应的问题，并让 OSD 跟随 Windows 明暗主题。

## 主要更新

- WebView2 返回 `0x80070490` 时不再在异常宿主进程内重试，改为执行一次受控进程重启。
- 恢复进程会等待旧进程完整退出后再初始化界面，避免两个实例同时占用 WebView2 和程序资源。
- 恢复链带一次性防循环标记；第二次仍失败时会停止自动重启并保留最终错误提示。
- 专用恢复退出路径可绕过“关闭到托盘”，但不会修改用户保存的关闭行为设置。
- 普通初始化超时仍保留一次同进程重试，DLL 缺失、程序集加载失败及其他 COM 错误继续按硬失败处理。
- OSD 的背景、边框、正文和非活动状态会跟随 Windows 浅色/深色主题，品牌强调色继续使用 `#D23D1D`。

## 发布包

- `MControl_v1.3.6_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.6_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.6` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 受 WebView2 卡死影响的用户升级后直接启动 M-Control 即可；第一次初始化若再次出现已知错误，程序会自动完成一次干净重启。
- 不需要主动卸载或重装 WebView2 Runtime；只有 v1.3.6 仍无法进入界面时，才重新采集诊断包。
- Windows 明暗主题切换后，后续 OSD 会自动使用对应配色。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

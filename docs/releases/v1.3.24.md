# M-Control v1.3.24

发布日期：2026-07-22

本版本修复部分机型在 CPU 风扇停转时，仪表盘固定误报约 1800 RPM 的问题。根因是 ThermalInfo/MSI ACPI 成功响应的状态字节被备用布局误当成了风扇转速高字节；现在明确的 0 RPM 会保持为 0，不再被覆盖为 1831 RPM。

## 主要更新

- 修复 ThermalInfo `Get_Fan(0)` 状态前缀响应的 CPU 风扇 RPM 解码。
- 响应首字节为 `0x01` 成功状态时，优先使用状态字节后的 RPM 布局。
- 首选布局明确解出 0 RPM 时直接保留停转状态，不再采用错误的备用非零结果。
- 修复按响应协议生效，不按 Vector 16 HX AI、`MS-15M3` 或单个 BIOS/EC 版本特判。
- 保持正常运行 RPM、EC 兜底、风扇曲线、校准和其他监控 Provider 的既有行为不变。

## 发布包

- `MControl_v1.3.24_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.24_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.24` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 验证结果

- 5 个正式前端脚本语法检查全部通过。
- 133 项 Node 静态与行为测试全部通过。
- 275 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- 新增 3 项 ThermalInfo RPM 回归测试，覆盖 8-byte/80-byte 停转响应和正常运行响应。
- 旧实现对停转样本稳定返回 1831；修复后返回 0，正常运行样本仍返回 1953。
- 正式安装包与便携包的 SHA256 已在本地重新计算并与清单一致。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 遇到 CPU 风扇停转时固定显示约 1800 RPM 的用户，升级后应显示 0 RPM；原反馈机型仍需实机复验。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

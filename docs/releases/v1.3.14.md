# M-Control v1.3.14

发布日期：2026-07-17

本版本统一仪表盘无数据状态：传感器尚未读到、读取失败或暂时不可用时保持问号占位，不再显示突兀的 `N/A`。

## 主要更新

- CPU 与 GPU 的温度、频率、功耗和风扇仪表在无有效数据时统一显示 `?`。
- 仪表曾经读到有效数值、随后读取失败时，也会恢复为 `?`，不会残留旧数值或切换为 `N/A`。
- RPM/百分比显示模式切换期间尚无监控数据时继续使用问号占位。
- 有效数据首次到达时仍保留现有的 `!` 过渡提示，随后正常显示数值。
- CPU 功耗不可用的具体原因仍保留在仪表悬停提示中，不影响故障判断。
- True Color、电池详情等其他页面中具有明确语义的 `N/A` 不受影响。

## 发布包

- `MControl_v1.3.14_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.14_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.14` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 验证结果

- 118 项 Node 静态与行为测试全部通过。
- 196 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- 无数据启动状态和读取成功后再次不可用两种路径均有回归门禁。
- 仪表盘脚本中已不存在直接显示 `N/A` 的更新入口。

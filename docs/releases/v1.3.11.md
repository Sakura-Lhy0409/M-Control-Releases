# M-Control v1.3.11

发布日期：2026-07-17

本版本修正两条灯效固件路线：Mystic Light PID_1801 的 A 面 Logo 不再混入键盘分区；SteelSeries 出厂红白响应会恢复固件满量程颜色增益，不再继承 GG 校色后显得偏暗。

## 主要更新

- PID_1801 的 24 区键盘与 A 面 Logo 按独立通道和独立 UI 状态处理。
- PID_1801 的恢复默认和 MCU 复位诊断能力按固件证据开放，不再被 24 区通用分支隐藏。
- SteelSeries“红白响应（固件）”部署前恢复 KLC 固件原生 `00 51 00 FF FF FF` 满量程颜色增益。
- Prism 与 Legacy 红白响应共用同一条已验证 KLC 命令；逐键红色、按键白色、200 ms 和响应 mask 保持不变。
- ALC 继续保留各机型 descriptor 的独立校色，不使用通用满量程覆盖。
- 新增两条 BUG 问题库记录，保留 PID_1801 分区和 Raider 18 Pro 红白响应亮度的证据、根因与回归门禁。

## 发布包

- `MControl_v1.3.11_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.11_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.11` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 验证结果

- 118 项 Node 静态与行为测试全部通过。
- 196 项 .NET 测试全部通过。
- Release 构建 0 警告、0 错误。
- Ghidra 固件证据确认 Raider KLC v2.61 的 Output selector `0x51` 会写入三通道颜色增益，并由运行态同步函数应用到全部键位。
- 字节级测试锁定红白响应的 65-byte `00 51 00 FF FF FF` 报文和发送顺序。
- PID_1801 当前仍需实机复验 Logo 分区与诊断按钮；Raider 18 Pro 仍需肉眼对比刚开机与重新部署后的红白响应亮度。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- Raider 18 Pro 用户应用“红白响应（固件）”后，应在 16/16 下获得接近固件首次启动状态的红色亮度。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。

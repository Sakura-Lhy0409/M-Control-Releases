# M-Control v1.3.19

发布日期：2026-07-20

本版本修复 SteelSeries 免驱基础路线在关机后无法恢复实时灯效的问题。`关闭（实时）`、纯色、逐键静态和软件动态效果不会写入 MCU Flash，完整断电后固件恢复自身灯效是正常行为；修复后，M-Control 再次启动时会把最后一次成功应用的易失状态重新写入硬件。

## 主要更新

- `steelseries-fw-basic` 成功加载本机已知状态后，会在 M-Control 启动时恢复最后一次实时灯效。
- `firmware-*` 和其他 MCU 持久效果仍由固件负责，不会在启动时重复写入。
- 首次安装且没有本机灯效记录时不会应用默认纯色，避免覆盖出厂灯效。
- “退出时保持灯效”仍只对支持的 Prism/native 实时路线开放，FW-basic 不会因为本修复而常驻后台。
- 节能模式的固件关灯守卫保持不变，节能模式仍优先于软件灯效恢复。

## 发布包

- `MControl_v1.3.19_Setup.exe`：普通安装版，可直接覆盖升级。
- `MControl_v1.3.19_Portable.zip`：普通便携版，完整解压后运行。
- `SHA256SUMS.txt`：正式产物 SHA256 校验清单。

`v1.3.19` 属于 hotfix，不提供内嵌 Fixed Version WebView2 Runtime 的大型安装包。

## 升级提示

- `v1.1.0` 或更高版本可直接运行安装包覆盖升级。
- 勾选“随 Windows 自动启动”后，登录系统时会自动恢复 FW-basic 易失灯效；未勾选时会在手动打开 M-Control 后恢复。
- 使用 `PID_1122` 键盘与 `PID_1161` 灯条组合、且未安装 SteelSeries GG 高级 KLC 通道的用户建议升级。
- 便携版不会自动覆盖自身文件，检查到新版本后会打开对应 Release 页面。
